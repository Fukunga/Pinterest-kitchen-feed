1. システム概要とマネタイズの仕組み

本システムは、かつてXServer等の有料サーバー上に構築されていたWordPressブログを完全に廃止し、維持費「0円」のGitHub Pagesに移行して構築された、Pinterestアフィリエイト全自動化システムです。

💡 マネタイズの核となる「直リンク（Direct Link）」方式

  • 従来の方式: Pinterest ➔ 自社WordPressブログ（記事内にアフィリンク） ➔ Amazon.com
  • 本システムの方式: Pinterest ➔ Amazon.com（アフィリエイト直リンク）

クッションページ（ブログ）を挟まないため、遷移時のユーザー離脱を完全に防止し、アソシエイト成約率を最大化します。WordPressサーバーの管理や、月々のサーバー維持費を完全ゼロ（0円）に削減しました。

⚠️ 規約違反を回避するための重要対策（広告開示）

AmazonアソシエイトおよびPinterestのポリシー上、アフィリエイト直リンクをソーシャルメディアへ投稿する場合は「広告であることを一目で視認可能に開示すること」が義務付けられています。

自動コンプライアンス処理 本システムの自動生成スクリプトは、Pinterestに読み込ませるRSSフィードの説明文（Description）の末尾に、自動的に #ad #affiliate ハッシュタグを挿入します。これによりアカウントBANのリスクを恒久的に回避します。

3. 技術仕様とシステム構造

プログラムはWindows上で動作し、生成した静的ファイルをGitHubリポジトリへ自動アップロード（デプロイ）することで機能します。

📂 プロジェクトのフォルダ構造マップ

🔗 過去のリンク（資産）を100%救済する仕組み

WordPressから移行した際、過去にPinterestに投稿された「古いピン」を踏んだユーザーがアクセスエラー（404エラー）にならないよう、以下の救済ルーティングを組んでいます。

  1. 日付ベースのパーマリンク（例: /2026/06/03/記事タイトル/）:
    対応するすべてのローカルフォルダ内に、meta refreshおよびJavaScriptを使用した「アフィリエイト自動転送HTML」を配置しています。
  2. パラメータベースの旧URL（例: /?p=25）:
    ルート直下の index.html にマッピング用のJavaScriptを組み込み、古い投稿ID（p値）から該当するAmazon商品ページへ瞬時にリダイレクトさせます。

4. 環境構築手順 (再セットアップ方法)

別のPCへ移行する場合や、環境を初期化して再起動する場合の手順です。

ステップ1: Python環境の準備

WindowsにPython 3.12以上がインストールされていることを確認し、プロジェクトフォルダ内でターミナル（PowerShell）を開いて以下を実行します。

pip install -r requirements.txt

ステップ2: 接続情報ファイル（.env）の作成

プロジェクトのルートディレクトリに .env というテキストファイルを作成し、以下の内容を記述します。

GEMINI_API_KEY=【Gemini API Key】
AMAZON_ASSOCIATE_TAG=saisaido-20

# GitHub自動パブリッシュ設定
GITHUB_USERNAME=fukunga
GITHUB_REPO=Pinterest-kitchen-feed
GITHUB_PAT=ghp_YOUR_PERSONAL_ACCESS_TOKEN_HERE
⚠️ セキュリティ上の重要注意 .env ファイルにはAPIキーと書き込み用権限を持つGitHubのトークンが含まれているため、絶対にリポジトリ（GitHub上）にアップロードしないでください。 必ず .gitignore に .env が指定されていることを確認してください。

ステップ3: タスクスケジューラへの登録

毎日決まった時間に自動でボットを動作させるため、Windowsのタスクスケジューラにタスクを設定します。
フォルダ内にある register_task.bat をダブルクリックして実行するだけで、自動的に毎日朝7:00にボットが起動するよう設定されます。

5. 日常運用と改善（PDCA）のポイント

自動運転中のパフォーマンス向上およびチェック方法です。

🔄 PDCAサイクルの回し方

  • Plan（商品候補の追加）:
    自動でトレンド商品を検知しますが、売りたい特定の商品がある場合は、data/products.csv の末尾に行を追加して、ASINと商品名、キーワードを追記しておくと、ボットが順次ピックアップして自動投稿します。
  • Do（自動稼働の監視）:
    時折、プロジェクトルートにある run_log.txt を開いてエラーが出ていないか、GitHub Pagesに最新のファイルがアップロードされているかを確認します。
  • Check（データの分析）:
    「AmazonアソシエイトUS」の管理画面で、クリックされた商品や売れ筋のジャンルを特定します。
  • Action（横展開）:
    クリックや売上の多い類似製品のASINを products.csv に優先的に追加して、アフィリエイト効率を高めます。

6. システムの撤退・削除手順

将来的にこのアフィリエイト事業を完全に停止（撤退）する場合、または元のWordPressブログに戻す場合のクリーンアップ手順です。

  1. Windowsの自動タスクの停止:
    管理者権限でコマンドプロンプト/PowerShellを開き、以下のコマンドを実行してWindowsから自動起動タスクを削除します。 
    schtasks /delete /tn "Pinterest_Amazon_Automation" /f
  2. GitHub Pagesの削除:
    GitHubリポジトリ（Pinterest-kitchen-feed）の **Settings ➔ General** の最下部にある「Danger Zone」から **「Delete this repository」** をクリックしてリポジトリごと削除します。これでWeb上のRSSや転送HTMLはすべて消去されます。
  3. DNSレコードの削除:
    エックスサーバー等のDNS管理画面から、追加したホスト名 kitchen（CNAME ➔ fukunga.github.io）のレコードを削除します。
  4. ローカルファイルの削除:
    PC上にあるプロジェクトフォルダ c:\Datas\Gemini\Pinterest_Amazon をそのままゴミ箱に入れて削除します。
